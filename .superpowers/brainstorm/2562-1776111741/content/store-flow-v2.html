Design Section 2 — Store Onboarding & Switcher Flow (updated)

Join requests require coordinator approval before access is granted

Schema change: store_memberships gains a status field

store_memberships.status TEXT — pending | active | rejected
// pending = awaiting coordinator approval
// active = full access granted
// rejected = coordinator declined

No separate join_requests table needed — memberships start as pending and activate on approval.
