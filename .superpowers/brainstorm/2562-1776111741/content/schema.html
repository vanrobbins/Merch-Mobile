Design Section 1 — Database Schema Changes

New tables + store_id added to all existing tables

New Tables

stores

id TEXT PRIMARY KEY
name TEXT NOT NULL
invite_code TEXT UNIQUE NOT NULL
created_at INTEGER NOT NULL
owner_uid TEXT NOT NULL

store_memberships

id TEXT PRIMARY KEY
store_id TEXT → stores.id
user_uid TEXT NOT NULL
role TEXT NOT NULL — coordinator | manager | staff
display_name TEXT NOT NULL
joined_at INTEGER NOT NULL

planogram_proposals

id TEXT PRIMARY KEY
planogram_id TEXT → planograms.id
store_id TEXT → stores.id
proposed_by_uid TEXT NOT NULL
proposed_at INTEGER NOT NULL
status TEXT — pending | approved | rejected
notes TEXT
slot_changes TEXT — JSON blob of slot diffs
reviewed_by_uid TEXT
reviewed_at INTEGER

Existing Tables — store_id FK added

Every existing table gets store_id TEXT NOT NULL REFERENCES stores(id) added via a Drift schema migration.

zones fixtures products planograms photo_docs

Active Store in App State

A activeStoreProvider (Riverpod) holds the currently selected store. All DAOs gain a watchByStore(storeId) method. The store switcher updates this provider — every screen reactively scopes to the active store.
