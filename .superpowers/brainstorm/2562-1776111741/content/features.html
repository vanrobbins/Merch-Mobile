Design Section 4 — Feature Work Summary

What gets wired up vs. what's new in v0.2

Feature Type What changes in v0.2
Zone Manager WIRE UP Zones persist to Drift (currently createDefaultZones() is in-memory only).
Full CRUD via ZonesDAO scoped to active store.
ZoneMapPainter loads from DB stream; zone count shown on Dashboard.
Floor Builder WIRE UP Fixtures save/load from FixturesDAO scoped to zone + store.
Add/move/rotate/delete all persist immediately.
Floor plan is read-only for staff (edit controls hidden by role).
Product Catalog WIRE UP Real CRUD: coordinator/manager can add, edit, delete products.
Products scoped to active store via ProductsDAO.
Staff browse + search read-only. Product count on Dashboard.
Planogram Editor WIRE UP Slot editing persists via PlanogramsDAO + PlanogramSlotsDAO.
Staff see a "Propose Change" button instead of direct edit.
Proposals stored in planogram_proposals table; manager/coordinator review queue.
Store & Members NEW Create store → auto invite code → coordinator role.
Join flow: enter code → pending → coordinator/manager approves → staff role assigned.
Members screen: pending queue + active members + role editing.
Multi-store switcher in app header.
Store Groups NEW Coordinator creates groups (e.g. "Gap Region 1").
Adds/removes stores from groups via a group management screen.
Groups displayed in the store switcher sheet for context.
Dashboard NEW Role-aware home screen replacing the current empty tab.
Coordinator: zone count, fixture count, product count, pending join requests, pending proposals, pending photo approvals.
Manager: pending proposals, pending photo approvals, product count.
Staff: own photo submissions, own proposals + their status.
Product → Planogram Assignment NEW In planogram detail view: slots show an "Assign Product" button.
Opens a searchable product picker (from catalog).
Selected product stored in planogram_slots.product_id.
Slot displays product name + SKU once assigned.
Staff submit this as a proposal; manager/coordinator edit directly.
Role Enforcement POLISH All edit controls (FABs, action buttons, context menus) check current user role before rendering.
GoRouter guards updated to block routes by role where needed.
Staff attempting restricted actions see an "Access restricted" snackbar.
