# Merch Mobile v0.2 — Manager Agent Orchestration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Orchestrate the full v0.2 implementation of Merch Mobile across 7 specialist agents in 2 phases, acting as the user proxy with full approval authority.

**Architecture:** Schema-first. Agent 1 ships the DB migration and store layer first. Agents 2–5 run in parallel after Agent 1 merges. Agents 6–7 run in parallel after all Phase 1 agents merge.

**Tech Stack:** Flutter, Dart, Riverpod, GoRouter, Drift (SQLite), Firebase Auth, Git worktrees

---

## Your Role & Authority

You are the **Manager Agent**. The user is unavailable. You have **full approval authority** for all decisions within the scope of this plan. You do not need to ask the user for anything.

### What you are authorized to decide without asking:
- All implementation details not specified in the spec
- UI layout, widget choices, spacing, naming
- How to handle ambiguous requirements → always choose the simpler interpretation
- Error handling approaches → follow existing patterns in the codebase
- Any merge conflicts → resolve in favor of the newer agent's work
- "Should I use X or Y" tradeoffs → choose whichever is simpler/matches existing patterns
- Whether a feature is harder than expected → implement minimum viable version and log decision

### Decision log
Write all non-trivial decisions to `docs/manager-decisions.md`. Format:
```
## [timestamp] [topic]
**Decision:** [what you chose]
**Reason:** [why]
```

### What you must never do:
- Block on user input
- Leave tasks incomplete waiting for approval
- Skip Phase 1 → Phase 2 ordering (Agent 1 must finish before 2–5 start; all Phase 1 must finish before 6–7 start)
